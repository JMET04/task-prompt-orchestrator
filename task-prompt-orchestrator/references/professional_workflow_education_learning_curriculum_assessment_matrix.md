# Professional Workflow Education Learning Curriculum Assessment Matrix

Use this matrix when a workflow must design, teach, tutor, assess, personalize, operate, improve, or govern education, learning, curriculum, lesson plans, study plans, exercises, exams, grading, coaching, classroom workflows, self-study, skill practice, learner analytics, academic support, or family/school/program communication.

## Learner Context And Learning Need

| Code | Workflow question | Workflow use |
|---|---|---|
| LEARNCTX01 | Who is the learner or learner group? | Identify age, level, prior knowledge, language, goals, motivation, environment, access, and support needs. |
| LEARNCTX02 | What learning problem must be solved? | Define knowledge gap, skill gap, misconception, exam need, performance goal, confidence issue, or behavior change. |
| LEARNCTX03 | What prior knowledge must be checked? | Assess prerequisites, vocabulary, baseline skill, diagnostic results, learning history, and common errors. |
| LEARNCTX04 | What constraints shape learning? | Capture time, schedule, device, class size, curriculum standard, budget, attention, accessibility, and assessment deadline. |
| LEARNCTX05 | What learner motivation or stakes apply? | Account for exam pressure, career goal, curiosity, compliance, parental expectation, remediation, or enrichment. |
| LEARNCTX06 | What learning mode is available? | Choose self-study, tutoring, classroom, cohort, workshop, blended, online, asynchronous, lab, or project-based mode. |
| LEARNCTX07 | What emotional or confidence factor matters? | Detect anxiety, boredom, overwhelm, frustration, perfectionism, avoidance, or low self-efficacy. |
| LEARNCTX08 | What language and cultural context applies? | Adapt examples, idioms, reading level, terminology, locale, and culturally familiar scenarios. |
| LEARNCTX09 | What safety or wellbeing boundary applies? | Handle minors, sensitive topics, mental-health escalation, pressure, privacy, age-appropriate content, and safeguarding. |
| LEARNCTX10 | What learner-context artifact is required? | Produce learner profile, diagnostic summary, study constraints note, readiness map, or support needs record. |

## Learning Objectives And Standards Alignment

| Code | Workflow question | Workflow use |
|---|---|---|
| OBJSTD01 | What should learners be able to do after learning? | Write observable objectives using action, content, condition, and quality standard. |
| OBJSTD02 | What standard or syllabus must be aligned? | Map to school standard, exam outline, competency model, job skill, certification, or course requirement. |
| OBJSTD03 | What cognitive level is required? | Classify recall, understanding, application, analysis, evaluation, creation, fluency, or transfer. |
| OBJSTD04 | What misconception should be targeted? | Identify wrong mental model, fragile vocabulary, procedural error, overgeneralization, or missing prerequisite. |
| OBJSTD05 | What mastery evidence proves the objective? | Define performance task, quiz, explanation, project, demonstration, problem set, rubric, or observed behavior. |
| OBJSTD06 | What objective sequence is logical? | Order prerequisites, core concepts, worked examples, guided practice, independent practice, and transfer tasks. |
| OBJSTD07 | What objective should be de-scoped? | Remove nice-to-have topics, advanced enrichment, off-exam material, or low-value repetition when time is tight. |
| OBJSTD08 | What objective needs differentiation? | Adjust goal depth for beginner, standard, advanced, special-needs, language learner, or accelerated learner. |
| OBJSTD09 | What objective risk exists? | Flag vague verbs, unmeasurable goals, misaligned assessment, missing standard, or overloaded lesson. |
| OBJSTD10 | What objective artifact is required? | Produce objective map, standards alignment table, competency matrix, exam blueprint, or mastery checklist. |

## Curriculum Architecture And Learning Path

| Code | Workflow question | Workflow use |
|---|---|---|
| CURRPATH01 | What curriculum unit or learning path is needed? | Define module, unit, course, bootcamp, tutoring plan, revision plan, or semester sequence. |
| CURRPATH02 | What progression should learners follow? | Sequence concepts from concrete to abstract, simple to complex, known to unknown, and practice to transfer. |
| CURRPATH03 | What pacing plan fits the constraints? | Allocate lesson time, homework, revision, breaks, assessment windows, and buffer for reteaching. |
| CURRPATH04 | What prerequisite bridge is needed? | Add refresher, vocabulary primer, mini-lesson, diagnostic task, or remedial path before core content. |
| CURRPATH05 | What spiral or retrieval structure applies? | Revisit key ideas through spaced repetition, interleaving, cumulative review, and mixed practice. |
| CURRPATH06 | What enrichment or extension path is needed? | Add challenge problems, projects, deeper readings, competitions, creative tasks, or independent inquiry. |
| CURRPATH07 | What remediation path is needed? | Plan reteaching, smaller steps, alternative explanation, worked examples, targeted practice, and confidence recovery. |
| CURRPATH08 | What learning resource map is needed? | Link readings, videos, examples, labs, worksheets, flashcards, simulations, tools, and reference materials. |
| CURRPATH09 | What path milestone should be checked? | Set checkpoints, formative assessments, portfolio reviews, parent/teacher review, or self-reflection stops. |
| CURRPATH10 | What curriculum artifact is required? | Produce course map, unit plan, study calendar, module sequence, learning path, or remediation plan. |

## Lesson And Activity Design

| Code | Workflow question | Workflow use |
|---|---|---|
| LESSOND01 | What lesson outcome should this session achieve? | Focus the lesson on one or a few measurable objectives, not a broad topic dump. |
| LESSOND02 | What hook or context will make the concept meaningful? | Use problem, story, demonstration, real-world case, question, artifact, or learner goal. |
| LESSOND03 | What explanation sequence is clearest? | Move through intuition, definition, example, counterexample, process, and transfer. |
| LESSOND04 | What activity structure should be used? | Choose direct instruction, inquiry, discussion, lab, project, debate, simulation, game, critique, or peer teaching. |
| LESSOND05 | What worked example is needed? | Provide annotated model, think-aloud, solved problem, exemplar, checklist, or before/after comparison. |
| LESSOND06 | What guided practice should follow? | Use scaffolded questions, partial solutions, prompts, hints, pairs, teacher checks, and gradual release. |
| LESSOND07 | What independent practice proves progress? | Assign problems, writing, demonstration, creation, reflection, project step, or retrieval task. |
| LESSOND08 | What closure consolidates learning? | Use exit ticket, summary, self-explanation, concept map, mini-quiz, or next-step preview. |
| LESSOND09 | What engagement risk should be managed? | Address passive listening, cognitive overload, off-task behavior, pacing mismatch, or unclear directions. |
| LESSOND10 | What lesson artifact is required? | Produce lesson plan, slide outline, activity sheet, teacher script, lab guide, or facilitation notes. |

| MATDEV07 | What differentiation variant is needed? | Produce easier, standard, challenge, language-supported, accessible, or extension versions. |
| MATDEV08 | What answer key or teacher note is required? | Add solution steps, common errors, marking guidance, discussion prompts, and extension ideas. |
| MATDEV09 | What material quality gate applies? | Check accuracy, alignment, clarity, cognitive load, accessibility, copyright, and visual readability. |
| MATDEV10 | What material artifact is required? | Produce worksheet, study guide, slide deck, handout, explainer, lab sheet, flashcard set, or answer key. |

## Practice Feedback And Skill Building

| Code | Workflow question | Workflow use |
|---|---|---|
| PRACTICE01 | What skill needs deliberate practice? | Target procedure, concept application, fluency, writing, speaking, problem solving, recall, or transfer. |
| PRACTICE02 | What practice type fits the skill? | Choose retrieval, spaced repetition, interleaving, drills, worked-example fading, simulation, peer practice, or project work. |
| PRACTICE03 | What difficulty progression is needed? | Sequence from easy to mixed, supported to independent, near transfer to far transfer, and single step to full task. |
| PRACTICE04 | What feedback should be immediate? | Provide correctness, explanation, hint, misconception correction, next step, encouragement, or rubric-aligned comment. |
| PRACTICE05 | What error pattern should be diagnosed? | Detect misconception, careless error, missing prerequisite, language issue, strategy mismatch, or test anxiety. |
| PRACTICE06 | What repetition is enough but not wasteful? | Set mastery threshold, spaced schedule, varied examples, and stop rule after consistent success. |
| PRACTICE07 | What learner reflection is useful? | Ask confidence rating, strategy note, error analysis, self-explanation, or next-practice choice. |
| PRACTICE08 | What adaptive hint sequence is needed? | Provide nudge, recall cue, worked step, partial answer, full explanation, and follow-up practice. |
| PRACTICE09 | What practice evidence should be logged? | Track attempts, time, accuracy, errors, hints used, confidence, mastery, and next target. |
| PRACTICE10 | What practice artifact is required? | Produce practice set, feedback bank, error analysis, adaptive hint plan, spaced review queue, or skill log. |

## Assessment Exam And Grading Design

| Code | Workflow question | Workflow use |
|---|---|---|
| ASSESS01 | What assessment purpose applies? | Choose diagnostic, formative, summative, exam prep, certification, placement, portfolio, or mastery check. |
| ASSESS02 | What blueprint aligns items to objectives? | Map topics, cognitive levels, weights, item types, time, difficulty, and standards coverage. |
| ASSESS03 | What item format fits the objective? | Use multiple choice, short answer, essay, oral response, performance task, project, lab, coding task, or critique. |
| ASSESS04 | What rubric is required? | Define criteria, levels, evidence, examples, point allocation, partial credit, and common error handling. |
| ASSESS05 | What fairness and accessibility checks apply? | Check language load, accommodations, bias, cultural assumptions, disability access, time limits, and scoring consistency. |
| ASSESS06 | What academic integrity control is needed? | Use item variants, open-book design, oral defense, process evidence, plagiarism check, proctoring, or honor policy. |
| ASSESS07 | What answer key or grading guide is needed? | Provide correct answers, explanations, alternate valid responses, scoring notes, and feedback comments. |
| ASSESS08 | What assessment data should be analyzed? | Review item difficulty, discrimination, topic gaps, error clusters, completion time, and learner confidence. |
| ASSESS09 | What retake or remediation rule applies? | Define revision, reassessment, partial retake, targeted support, grade policy, and mastery recovery path. |
| ASSESS10 | What assessment artifact is required? | Produce exam blueprint, quiz, rubric, grading guide, item bank, feedback report, or remediation plan. |

## Tutoring Coaching And Personalized Support

| Code | Workflow question | Workflow use |
|---|---|---|
| TUTOR01 | What individualized learning goal should drive tutoring? | Set target skill, exam outcome, confidence goal, assignment support, or remediation objective. |
| TUTOR02 | What diagnostic should start the session? | Use warm-up question, mini-quiz, conversation, sample work review, or think-aloud problem. |
| TUTOR03 | What tutoring stance fits the learner? | Choose Socratic, direct explanation, coaching, motivational, exam-strategy, metacognitive, or remedial stance. |
| TUTOR04 | What question should uncover thinking? | Ask learner to explain steps, predict, compare, justify, spot error, or choose strategy. |
| TUTOR05 | What scaffold should be removed gradually? | Move from modeling to prompts, hints, partial independence, independent solution, and transfer challenge. |
| TUTOR06 | What motivational support is appropriate? | Use specific praise, growth framing, manageable goals, anxiety reduction, progress evidence, and choice. |
| TUTOR07 | What boundary should tutoring respect? | Avoid doing graded work for the learner, unsafe advice, age-inappropriate content, privacy breach, or overdependency. |
| TUTOR08 | What parent/teacher update is needed? | Share progress, next practice, concerns, support request, and boundaries without overexposing private details. |
| TUTOR09 | What between-session work should be assigned? | Give short practice, retrieval plan, reflection, reading, project step, or exam drill with clear done criteria. |
| TUTOR10 | What tutoring artifact is required? | Produce session plan, tutoring script, learner progress note, homework plan, or coaching feedback. |

## Learning Analytics And Progress Monitoring

| Code | Workflow question | Workflow use |
|---|---|---|
| LEARNMET01 | What progress metric matters? | Track mastery, accuracy, fluency, retention, assignment completion, engagement, confidence, attendance, or performance. |
| LEARNMET02 | What diagnostic trend should be monitored? | Identify error clusters, forgotten prerequisites, slow pacing, disengagement, anxiety signals, or topic gaps. |
| LEARNMET03 | What data source is available? | Use quiz results, LMS logs, homework, tutor notes, observations, surveys, portfolios, or exam results. |
| LEARNMET04 | What denominator prevents misleading progress? | Define cohort, learner group, item pool, attempts, time window, standard, and opportunity to learn. |
| LEARNMET05 | What early-warning threshold triggers support? | Set triggers for low score, missed work, repeated errors, low confidence, absence, or sudden decline. |
| LEARNMET06 | What progress visualization is useful? | Use mastery map, topic heatmap, timeline, practice queue, class dashboard, or learner report. |
| LEARNMET07 | What privacy boundary applies to learning data? | Limit sharing of grades, minors' data, accommodations, behavior notes, and sensitive context. |
| LEARNMET08 | What decision follows the data? | Choose reteach, advance, assign practice, schedule tutoring, contact guardian, adjust pacing, or revise material. |
| LEARNMET09 | What learner-facing feedback should be shown? | Provide strengths, next target, practice plan, confidence note, and evidence of improvement. |
| LEARNMET10 | What analytics artifact is required? | Produce progress report, mastery dashboard, error analysis, intervention list, or learning data note. |

## Family Stakeholder And Institutional Communication

|---|---|---|

| EDUSTAKE03 | What privacy and consent boundary applies? | Respect FERPA-like rules, school policy, minors' data, consent, confidentiality, and learner dignity. |
| EDUSTAKE04 | What communication cadence is appropriate? | Set weekly updates, milestone reports, exam alerts, incident notices, office hours, or end-of-unit summaries. |
| EDUSTAKE05 | What concern needs escalation? | Route academic risk, wellbeing issue, absence, behavior, safety, access barrier, or conflict to the right owner. |
| EDUSTAKE06 | What shared action plan is needed? | Assign learner practice, parent support, teacher adjustment, tutor session, accommodations, or resource access. |
| EDUSTAKE07 | What tone should communication use? | Be specific, respectful, strengths-based, actionable, age-appropriate, culturally aware, and non-shaming. |
| EDUSTAKE08 | What evidence should be included? | Cite work samples, scores, observations, attendance, learner reflection, and progress against objectives. |
| EDUSTAKE09 | What response or approval is needed? | Request meeting, consent, resource, schedule change, accommodation review, or decision. |
| EDUSTAKE10 | What communication artifact is required? | Produce parent update, learner feedback note, teacher handoff, progress email, meeting agenda, or intervention plan. |

## Learning Quality Review And Iteration

| Code | Workflow question | Workflow use |
|---|---|---|
| EDUQUAL01 | What quality standard should the learning design meet? | Check alignment, clarity, engagement, accuracy, accessibility, assessment validity, and learner progress. |
| EDUQUAL02 | What learner evidence should drive revision? | Use scores, work samples, error patterns, feedback, engagement, confidence, attendance, and transfer performance. |
| EDUQUAL03 | What instructor or tutor feedback matters? | Capture pacing, confusion points, material gaps, behavior issues, facilitation notes, and support burden. |
| EDUQUAL04 | What assessment validity issue exists? | Check if test items measure objectives, avoid trick wording, balance difficulty, and produce reliable scores. |
| EDUQUAL05 | What content accuracy review is required? | Verify facts, formulas, sources, standards, answer keys, examples, and copyright. |
| EDUQUAL06 | What revision should happen first? | Prioritize blockers, misconceptions, inaccessible materials, broken assessments, low engagement, and high-impact gaps. |
| EDUQUAL07 | What versioning is needed? | Track material version, curriculum changes, assessment updates, learner cohort, and retired content. |
| EDUQUAL08 | What pilot or A/B learning test is useful? | Compare examples, practice sequence, feedback style, lesson format, or pacing with defined success metrics. |
| EDUQUAL09 | What reusable lesson should be captured? | Save effective explanation, example, misconception fix, assessment item, scaffold, and coaching strategy. |
| EDUQUAL10 | What quality artifact is required? | Produce learning review, revision log, QA checklist, pilot report, curriculum update, or lesson-retro note. |
