# Professional Workflow Customer Experience Journey Matrix

Use this when a workflow affects users, customers, buyers, learners, clients, patients, citizens, operators, or any audience moving through a journey from discovery to adoption, support, retention, expansion, or exit. Customer experience journey rules define touchpoints, user goals, pain points, evidence, service handoffs, and improvement actions.

Selection rule:

1. Identify the user/customer journey stage and audience.
2. Select the closest journey code.
3. Add persona, trigger, touchpoint, user goal, pain point, evidence, owner, and next action to the prompt packet.
4. Do not optimize a journey step without naming how it affects user outcome, business outcome, and downstream support.
