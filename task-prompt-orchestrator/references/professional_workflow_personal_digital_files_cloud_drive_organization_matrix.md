# Professional Workflow Personal Digital Files Cloud Drive Organization Matrix

Use this matrix when a workflow must manage personal digital files, downloads, desktop clutter, cloud drives, folder structure, file naming, duplicates, backups, archives, sharing links, privacy, migration, or recurring file maintenance. Use this for personal organization, retrieval, backup, sharing hygiene, and storage cleanup, not to replace enterprise records management, legal discovery, cybersecurity incident response, professional digital asset management, or regulated data-retention advice.

## DIGSCP - Digital File Scope

| Code | Workflow question | Workflow use |
|---|---|---|
| DIGSCP01 | What file collection is in scope? | Define downloads, desktop, documents, cloud drive, project folder, school files, receipts, exports, or old device files. |
| DIGSCP02 | Why is file organization needed now? | Link to full storage, hard-to-find files, device migration, tax season, project closeout, sharing risk, or clutter. |
| DIGSCP03 | Who owns or uses the files? | Identify self, household, family member, student, caregiver, collaborator, or shared account. |
| DIGSCP04 | What outcome is desired? | Choose clean desktop, searchable folder, archive, backup, migration, shared packet, or deletion plan. |
| DIGSCP05 | What sensitivity applies? | Flag identity, finance, health, legal, children, work-adjacent, private photos, or credentials. |
| DIGSCP06 | What should be excluded? | Exclude legal discovery, enterprise records retention, forensic handling, regulated compliance, and incident response. |
| DIGSCP07 | What time range matters? | Scope today, month, year, project, school term, old computer, cloud account, or full archive. |
| DIGSCP08 | What systems are involved? | Map computer, phone, cloud drive, external disk, email attachments, messaging apps, and backup service. |
| DIGSCP09 | What record should be kept? | Track source, destination, backup status, deletion decisions, shared links, and archive location. |
| DIGSCP10 | What scope artifact is required? | Produce digital file brief, source map, cleanup plan, or archive checklist. |

## NAMFIL - Naming Dates And Versions

| Code | Workflow question | Workflow use |
|---|---|---|
| NAMFIL01 | What naming convention is needed? | Use date, topic, person, source, version, status, and file type based on retrieval need. |
| NAMFIL02 | What date format should be used? | Prefer sortable dates such as YYYY-MM-DD when files need chronological order. |
| NAMFIL03 | What version label is needed? | Mark draft, final, signed, submitted, paid, archived, original, edited, or superseded. |
| NAMFIL04 | What filename is too vague? | Rename scan, download, image, final-final, untitled, or random export names. |
| NAMFIL05 | What filename should not expose private data? | Avoid unnecessary SSN, full DOB, medical details, child details, or passwords in filenames. |
| NAMFIL06 | What batch rename is safe? | Rename low-risk photos, receipts, scans, and project exports after backup or sample test. |
| NAMFIL07 | What original name should be preserved? | Keep legal, official, signed, vendor, or system-generated names when changing could confuse records. |
| NAMFIL08 | What version conflict exists? | Resolve duplicate finals, edited copies, old templates, and latest-approved confusion. |
| NAMFIL09 | What naming exception is allowed? | Keep vendor invoice numbers, official form IDs, and platform filenames when useful. |
| NAMFIL10 | What naming artifact is required? | Produce naming guide, version policy, or rename queue. |

## DEDUPF - Duplicates And Large File Cleanup

| Code | Workflow question | Workflow use |
|---|---|---|
| DEDUPF01 | What counts as a duplicate? | Distinguish exact duplicate, old version, export copy, backup copy, attachment copy, and compressed copy. |
| DEDUPF02 | What duplicate should be kept? | Keep highest quality, official, signed, most recent, most complete, or best metadata copy. |
| DEDUPF03 | What duplicate should be deleted? | Delete redundant downloads, repeated exports, old installers, copied folders, and obvious duplicates after backup. |
| DEDUPF04 | What large files drive storage use? | Find videos, disk images, zips, installers, datasets, exports, backups, and app libraries. |
| DEDUPF05 | What large file should be archived? | Move rare-use but valuable files to external drive, archive folder, or cold storage. |
| DEDUPF06 | What large file should be compressed? | Compress completed project folders, logs, scans, or media only when usability remains acceptable. |
| DEDUPF07 | What file needs manual review? | Review official records, sentimental files, signed documents, and family archives before deletion. |
| DEDUPF08 | What cleanup tool can help? | Use built-in storage view, duplicate finder, file manager, or script with cautious sampling. |
| DEDUPF09 | What deletion recovery exists? | Confirm trash/recycle bin, backup, cloud version history, and undo window. |
| DEDUPF10 | What cleanup artifact is required? | Produce duplicate policy, deletion log, storage-savings note, or review queue. |
