# Professional Workflow Personal Errands Local Task Coordination Matrix

Use this matrix when a workflow must manage personal errands, local task coordination, pickups, dropoffs, returns, mail or shipping, dry cleaning, pharmacy pickup, branch or counter visits, route batching, delegated errands, receipts, proof records, or failed-errand recovery. Use this for local coordination, records, and planning, not to replace legal advice, medical advice, financial advice, emergency dispatch, regulated courier work, or professional procurement.

## ERRSCP - Errand Scope

| Code | Workflow question | Workflow use |
|---|---|---|
| ERRSCP01 | What errand is in scope? | Define pickup, dropoff, return, mail, payment, branch visit, document submission, repair pickup, or local service task. |
| ERRSCP02 | Why must the errand happen now? | Link to deadline, return window, appointment, household need, travel, event, bill due date, or convenience batch. |
| ERRSCP03 | Who benefits from the errand? | Identify self, family member, roommate, elder, child, pet, neighbor, friend, or household. |
| ERRSCP04 | What outcome proves completion? | Define item obtained, item delivered, receipt captured, form submitted, confirmation received, or issue escalated. |
| ERRSCP05 | What constraints apply? | Note hours, location, transport, weather, mobility, budget, wait time, ID, authorization, or storage limits. |
| ERRSCP06 | What should be excluded? | Exclude emergency response, legal representation, medical decisions, financial advice, regulated courier work, or unsafe errands. |
| ERRSCP07 | What priority level applies? | Classify urgent, deadline-bound, high-value, perishable, optional, batchable, or deferrable. |
| ERRSCP08 | What authority is needed? | Check ID, account access, pickup code, authorization letter, consent, payment method, or appointment holder. |
| ERRSCP09 | What record should be kept? | Track receipts, photos, confirmation numbers, service tickets, names, timestamps, or failed-attempt notes. |
| ERRSCP10 | What scope artifact is required? | Produce errand brief, local task list, route packet, or completion checklist. |

## ERRLST - Errand List And Priorities

| Code | Workflow question | Workflow use |
|---|---|---|
| ERRLST01 | What errands are on the list? | Capture every stop, item, person, address, deadline, opening hour, and required proof. |
| ERRLST02 | Which errand is time-critical? | Prioritize closing times, return windows, appointment slots, school pickup, prescriptions, or perishables. |
| ERRLST03 | Which errand has high consequence? | Flag costly items, official documents, account access, health-related pickups, or landlord/agency deadlines. |
| ERRLST04 | Which errand can be batched? | Group nearby stops, similar documents, shared payment methods, or same-store tasks. |
| ERRLST05 | Which errand can be deferred? | Move optional, low-value, flexible, or online-replaceable tasks to later. |
| ERRLST06 | What errand depends on another? | Sequence label printing before shipping, cash withdrawal before payment, pickup before dropoff, or form copy before submission. |
| ERRLST07 | What errand needs preparation? | Gather package, receipt, ID, documents, bags, measurements, photos, codes, or contact numbers. |
| ERRLST08 | What errand should be converted online? | Replace in-person tasks with online form, delivery, pickup locker, mail-in return, or call when suitable. |
| ERRLST09 | What errand is no longer needed? | Remove duplicates, expired return windows, solved tasks, wrong locations, or tasks assigned to someone else. |
| ERRLST10 | What list artifact is required? | Produce prioritized errand list, dependency map, or today/this-week split. |

## ROUTER - Route And Time Blocking

| Code | Workflow question | Workflow use |
|---|---|---|
| ROUTER01 | What route should be used? | Order stops by location, hours, dependencies, traffic, parking, transit, weather, or load. |
| ROUTER02 | What time window fits each stop? | Match opening hours, appointment windows, pickup cutoffs, school schedules, or quiet hours. |
| ROUTER03 | What travel mode is best? | Choose walking, driving, bike, transit, rideshare, delivery service, or delegation based on constraints. |
| ROUTER04 | What parking or access issue exists? | Check loading zone, building entry, elevator, curbside pickup, locker, security desk, or accessibility route. |
| ROUTER05 | What route risk should be avoided? | Avoid unsafe areas, severe weather, heavy items on foot, tight transfers, or closing-time dependency chains. |
| ROUTER06 | What buffer is needed? | Add wait time, traffic, line time, packing time, elevator time, and contingency for delays. |
| ROUTER07 | What item handling order matters? | Do cold, fragile, bulky, private, or high-value items in an order that preserves safety and condition. |
| ROUTER08 | What route can be split? | Divide by neighborhood, day, helper, vehicle availability, or priority when the list is too large. |
| ROUTER09 | What stop can be skipped if late? | Set drop rules for optional, low-value, or non-deadline errands. |
| ROUTER10 | What route artifact is required? | Produce route plan, stop order, time block, or map-ready checklist. |

| DELEGA09 | How should thanks or repayment happen? | Plan reimbursement, favor return, gratitude message, tip, or shared household credit. |
| DELEGA10 | What delegation artifact is required? | Produce delegate brief, permission note, proof checklist, or reimbursement note. |
