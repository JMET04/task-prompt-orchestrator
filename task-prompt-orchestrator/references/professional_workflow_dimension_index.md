## Recommended Assembly Order

Use this order for serious workflow design:

1. **Goal and base route**: choose base professional workflow from `professional_workflow_library.md`.
2. **Specific case**: choose specialized subworkflow from subworkflow and scenario expansion matrices.
3. **Capability discovery**: scan available skills, plugins, connectors, tools, local scripts, and built-in abilities; map triggers, fit, limits, fallback, route plan, evidence, and audit before inventing new work.
4. **Plugin-skill scan/auto-install**: scan visible session capabilities, local skills, plugin cache, dynamic tools, GitHub/Codex/OpenAI directories, sparse-plugin gaps, exact candidates, install intent, request flow, post-install re-scan, fallback, auto-use policy, and audit before claiming plugin capability or requesting installation.
