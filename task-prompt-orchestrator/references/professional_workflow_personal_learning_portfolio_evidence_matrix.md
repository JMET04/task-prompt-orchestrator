# Professional Workflow Personal Learning Portfolio Evidence Matrix

Use this matrix when a workflow must manage self-study, skill goals, learning resources, practice logs, evidence capture, portfolios, feedback, mentors, certificates, learning habits, public proof, or learning review. Use this for organization, planning, evidence, and reflection, not to replace school policy, credentialing authorities, academic integrity rules, formal assessment, therapy, or licensed coaching.

## LRNSCP - Learning Portfolio Scope

| Code | Workflow question | Workflow use |
|---|---|---|
| LRNSCP01 | What learning area is in scope? | Define subject, skill, role capability, hobby, technical area, language, creative practice, or professional competency. |
| LRNSCP02 | Why is this learning important now? | Link learning to career move, project need, school goal, exam, personal growth, portfolio, or life transition. |
| LRNSCP03 | What level is the learner starting from? | Record beginner, rusty, intermediate, advanced, switching fields, or uneven skill profile. |
| LRNSCP04 | What level is good enough? | Define minimum usable ability, portfolio-ready proof, exam readiness, job-readiness, or personal satisfaction. |
| LRNSCP05 | What time horizon applies? | Set sprint, semester, quarter, year, bootcamp, project window, or open-ended cadence. |
| LRNSCP06 | What constraints shape the plan? | Note time, budget, language, access, equipment, attention, health, work schedule, or caregiving limits. |
| LRNSCP07 | What learning format fits? | Choose self-study, course, project, apprenticeship, tutor, cohort, reading group, practice routine, or mixed path. |
| LRNSCP08 | What evidence standard is needed? | Decide whether proof must satisfy self-review, mentor review, employer, client, school, platform, or credential body. |
| LRNSCP09 | What boundaries must be respected? | Avoid cheating, false credential claims, unsafe practice, privacy leaks, copied work, or unlicensed advice. |
| LRNSCP10 | What scope artifact is required? | Produce learning brief, portfolio scope note, competency map, or study charter. |

## GOALRN - Learning Goals And Outcomes

| Code | Workflow question | Workflow use |
|---|---|---|
| GOALRN01 | What outcome should the learner demonstrate? | Define observable skill, finished artifact, solved problem, exam score, presentation, or performance. |
| GOALRN02 | What subskills make up the goal? | Break the target into concepts, techniques, tools, habits, vocabulary, judgment, and execution steps. |
| GOALRN03 | What milestone sequence is realistic? | Arrange beginner foundations, guided practice, independent practice, feedback, refinement, and public proof. |
| GOALRN04 | What success criteria are measurable? | Use accuracy, speed, depth, rubric score, artifact quality, repetitions, publication, or reviewer signoff. |
| GOALRN05 | What anti-goals should be excluded? | Avoid vague mastery, endless course watching, credential chasing, vanity projects, or overbroad scope. |
| GOALRN06 | What transfer context matters? | Connect learning to real work, portfolio projects, interviews, teaching, personal life, or community contribution. |
| GOALRN07 | What prerequisite gap must be closed? | Identify math, language, tooling, domain basics, typing, reading, setup, or terminology gaps. |
| GOALRN08 | What stretch goal is optional? | Mark advanced artifact, competition, public talk, open-source contribution, client-ready demo, or certificate. |
| GOALRN09 | What goal conflict exists? | Resolve speed versus depth, breadth versus specialization, theory versus practice, or private versus public proof. |
| GOALRN10 | What goal artifact is required? | Produce goal tree, outcome rubric, milestone plan, or competency checklist. |

## CURMAP - Curriculum And Resource Map

| Code | Workflow question | Workflow use |
|---|---|---|
| CURMAP01 | What resources are already available? | Inventory books, courses, docs, papers, videos, mentors, examples, tools, datasets, labs, or communities. |
| CURMAP02 | Which resources are authoritative? | Prefer official docs, primary sources, reputable instructors, proven textbooks, standards, or domain references. |
| CURMAP03 | What learning path should be followed? | Sequence foundations, examples, guided exercises, projects, deliberate practice, review, and proof. |
| CURMAP04 | What resource should be skipped? | Remove duplicate courses, outdated materials, low-quality advice, distracting content, or unsafe shortcuts. |
| CURMAP05 | What resource format fits the learner? | Choose reading, video, interactive lab, workbook, flashcards, project, mentor call, or practice platform. |
| CURMAP06 | What tool setup is required? | Prepare software, accounts, workspace, notebooks, repositories, templates, instruments, or practice materials. |
| CURMAP07 | What source gaps remain? | Identify missing examples, datasets, feedback access, assessment rubrics, official rules, or advanced references. |
| CURMAP08 | What resource budget applies? | Set spend limits for books, courses, subscriptions, hardware, tutoring, exams, or memberships. |
| CURMAP09 | How should resources be indexed? | Tag by topic, level, format, source authority, date, completion state, and evidence produced. |
| CURMAP10 | What curriculum artifact is required? | Produce resource map, syllabus, reading queue, setup checklist, or study path board. |

## STUDYS - Study Session Planning

| Code | Workflow question | Workflow use |
|---|---|---|
| STUDYS01 | What is the next study session objective? | Define one concept, exercise set, chapter, lab, draft, review, or practice block. |
| STUDYS02 | How much time is available? | Fit session to 15 minutes, deep-work block, commute, evening, weekend, or recurring calendar slot. |
| STUDYS03 | What input is needed before starting? | Gather notes, resources, tools, files, examples, assignment prompts, or previous feedback. |
| STUDYS04 | What active practice will happen? | Include recall, problem solving, writing, coding, drawing, speaking, building, testing, or teaching. |
| STUDYS05 | What passive consumption is allowed? | Limit reading or video to a clear question, timestamp, chapter, or setup explanation. |
| STUDYS06 | How should distractions be controlled? | Set device mode, workspace, timer, app blockers, checklist, or social boundary. |
| STUDYS07 | What should be captured during the session? | Record notes, mistakes, examples, screenshots, commits, sketches, audio, or solved exercises. |
| STUDYS08 | What completion rule applies? | Stop when objective is met, time ends, confusion is logged, artifact exists, or feedback question is ready. |
| STUDYS09 | What recovery plan handles missed sessions? | Reschedule, shrink scope, review blockers, reduce streak pressure, or restart from a smaller task. |
| STUDYS10 | What session artifact is required? | Produce session plan, study note, practice output, confusion log, or next-action card. |

## PRACLG - Practice Log And Repetition

| Code | Workflow question | Workflow use |
|---|---|---|
| PRACLG01 | What practice unit should be repeated? | Track flashcards, problems, drills, sketches, reps, code kata, pronunciation, cases, or mock tasks. |
| PRACLG02 | What repetition schedule is appropriate? | Use daily, spaced repetition, weekly review, sprint cycles, deliberate practice blocks, or maintenance cadence. |
| PRACLG03 | What error types should be logged? | Capture concept gaps, careless mistakes, tool friction, timing issues, vocabulary gaps, or judgment errors. |
| PRACLG04 | What feedback should modify repetition? | Increase hard items, retire mastered items, revisit forgotten topics, or change practice method. |
| PRACLG05 | What evidence proves practice occurred? | Keep timestamps, solved sets, commits, recordings, sketches, screenshots, quiz results, or peer notes. |
| PRACLG06 | What quality signal matters? | Track accuracy, fluency, speed, independence, explanation quality, transfer, or reduced error rate. |
| PRACLG07 | What boredom or plateau signal appears? | Detect stalled progress, avoidance, shallow repetition, low challenge, or resource fatigue. |
| PRACLG08 | How should practice difficulty change? | Add constraints, mix topics, increase complexity, teach back, simulate real context, or seek harder feedback. |
| PRACLG09 | What practice data should stay private? | Protect sensitive reflections, recordings, health data, client examples, schoolwork, or personal identifiers. |
| PRACLG10 | What practice artifact is required? | Produce practice log, repetition tracker, mistake bank, drill plan, or progress chart. |

## EVIDLR - Evidence And Artifact Capture

| Code | Workflow question | Workflow use |
|---|---|---|
| EVIDLR01 | What learning evidence should be captured? | Collect projects, notes, tests, reflections, recordings, screenshots, certificates, reviews, or shipped work. |
| EVIDLR02 | What evidence is strongest? | Prefer original artifacts, before-after comparisons, measurable results, reviewer feedback, and reproducible work. |
| EVIDLR03 | What context should each artifact include? | Add goal, date, role, tools, constraints, process, source materials, and what changed. |
| EVIDLR04 | What evidence needs permission? | Check client work, school assignments, employer materials, private messages, images, recordings, and third-party IP. |
| EVIDLR05 | What evidence should be anonymized? | Remove names, contact info, faces, account IDs, grades, medical details, or proprietary data. |
| EVIDLR06 | What version history matters? | Keep drafts, commits, revisions, review notes, failed attempts, and final version when they show growth. |
| EVIDLR07 | What proof needs external verification? | Link to live demos, repositories, certificates, publications, badges, competition results, or reviewer statements. |
| EVIDLR08 | How should evidence be stored? | Use folder, repository, portfolio CMS, spreadsheet, note system, drive, or secure archive. |
| EVIDLR09 | What evidence should be excluded? | Remove low-quality, unverifiable, copied, misleading, outdated, private, or irrelevant artifacts. |
| EVIDLR10 | What evidence artifact is required? | Produce evidence packet, artifact inventory, proof table, or portfolio source folder. |

| PORTLR09 | How should the portfolio be maintained? | Set update cadence, archive rules, broken-link checks, new artifact intake, and outdated claim cleanup. |
| PORTLR10 | What portfolio artifact is required? | Produce portfolio outline, case study, proof deck, repo README, or publish-ready portfolio packet. |

## FEEDLR - Feedback And Review

| Code | Workflow question | Workflow use |
|---|---|---|
| FEEDLR01 | Who should review the work? | Choose teacher, mentor, peer, domain expert, user, client, community, evaluator, or self-review rubric. |
| FEEDLR02 | What feedback question should be asked? | Ask about correctness, clarity, depth, style, usability, originality, portfolio strength, or next practice. |
| FEEDLR03 | What rubric should guide feedback? | Use criteria for accuracy, reasoning, craft, communication, completeness, evidence, and transfer. |
| FEEDLR04 | What feedback format is best? | Request comments, score, annotated artifact, call notes, screen recording, checklist, or pull request review. |
| FEEDLR05 | What feedback should be ignored? | Filter vague opinions, misaligned preferences, low-expertise advice, unsafe suggestions, or off-goal criticism. |
| FEEDLR06 | What feedback pattern repeats? | Detect recurring gaps in fundamentals, clarity, speed, polish, originality, or confidence. |
| FEEDLR07 | What revision should follow? | Convert feedback into one change, practice block, resource lookup, rewrite, retest, or portfolio update. |
| FEEDLR08 | What feedback needs a second opinion? | Escalate conflicting, high-stakes, domain-specific, legal, medical, or credential-related feedback. |
| FEEDLR09 | How should feedback be acknowledged? | Thank reviewer, confirm interpretation, ask follow-up, log decision, and share revision when appropriate. |
| FEEDLR10 | What feedback artifact is required? | Produce review log, rubric scorecard, revision plan, or feedback-to-action table. |
