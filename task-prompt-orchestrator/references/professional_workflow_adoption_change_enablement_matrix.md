# Professional Workflow Adoption and Change Enablement Matrix

Use this when a workflow, prompt library, product, tool, process, policy, template, automation, or operating model must be adopted by real users after it is designed or delivered. Adoption and change enablement rules define readiness, rollout, communications, training, migration, resistance handling, behavior reinforcement, support, measurement, and sustainment.

Selection rule:

1. Identify what is changing and who must adopt it.
2. Select the closest adoption/change enablement code.
3. Add affected audience, readiness state, rollout plan, training, support, migration, resistance, measurement, and sustainment evidence to the prompt packet.
4. Do not call a change successful until target users can perform the new behavior and adoption evidence exists.
